myFlix: A Netflix-like Streaming Platform

Case Study for Developer Portfolio

Case Study


Overview

The primary purpose of myFlix is to provide users with a seamless and enjoyable movie-watching experience. The platform offers features such as user registration and a personalized user profile. Users can save their favorite films for later viewing.

Objection

  • User Registration and Authentication: Allow users to create accounts, log in securely, and manage their profiles.
  • Movie Database: Implement a comprehensive movie database that includes movie details.
  • User Interactions: Enable users to save their favorite movies to a personal list.
  • Responsive Design: Ensure the website is accessible and visually appealing on both desktop and mobile devices.
  • Seamless Backend Integration: Connect the React.js frontend with the backend API to ensure smooth data retrieval and updates.

Approach

The development of the myFlix website involved the following steps:


  • Frontend Development: React.js was chosen as the frontend framework due to its flexibility and efficiency. The developer designed the user interface, implemented user registration/authentication, and created components for browsing movies and managing user profiles.
  • Backend Integration: To connect the front-end with the backend, an API was developed using Node.js and Express.js. This API served as the bridge for data retrieval and updates. Secure authentication and authorization mechanisms were implemented to protect user data.
  • Database Management: MongoDB was selected as the database system to store movie information and user profiles. The database was structured to support efficient data retrieval.
  • Deployment: The myFlix website was deployed on a Netlify platform, ensuring scalability and availability to a global audience.

Challenges

Authentication and Authorization

  • Problem: Ensuring secure user authentication and authorization was crucial to protect user data and sensitive operations. Users needed to have their identities confirmed and their actions properly controlled based on their roles.
  • Challenges:
    • • Safeguarding user passwords and sensitive data.
    • • Balancing user convenience with security.
    • • Managing user sessions and token expiration.
  • Decisions Made:
    • • Employed bcrypt hashing to securely store and verify user passwords.
    • • Implemented token-based authentication with JWTs, providing a balance between security and ease of implementation.

Cross-Origin Resource Sharing (CORS)

  • Problem: Configuring CORS settings to allow secure cross-origin requests was necessary for the front-end to communicate with the backend. This was crucial because the front-end and back-end might reside in different domains.
  • Challenges:
    • • Allowing cross-origin requests without compromising security.
    • • Ensuring that only trusted domains could access the API.
  • Decisions Made:
    • • Configured CORS settings on the backend server to specify which origins were allowed to access the API.
    • • Set up the API to respond with appropriate CORS headers based on the incoming requests' origins.
    • • Implemented strict CORS policies in production to limit potential security risks.
